## 二、遇到的问题

### 2.1 问题1：chunk质量问题
**现象**: 整个PDF被解析成1个巨大的chunk（one模式），检索时所有问题都命中同一个chunk，LLM被大量无关内容干扰。

**原因**: 使用了"one"解析模式，整个文档只有1个chunk。

### 2.2 问题2：特殊字符丢失
**现象**: VLM正确输出了"flow first passes through perforated disk 6'', then perforated disk 6', then perforated disk 6"，但存到ES后变成了"flow first pas through perfor disk 6 then perfor disk 6 then perfor disk 6"，`6''` `6'`的上标符号全丢了。

**原因**: ES在索引时对特殊字符进行了标准化处理。

### 2.3 问题3：VLM描述不准确
**现象**: VLM描述图片时，先说整体结构（入口→锥形部分→外壳），后说内部小部件（配气盘顺序），导致LLM回答时跟着VLM的顺序走。

**原因**: VLM prompt没有明确要求先描述内部小部件。

### 2.4 问题4：部件位置关系描述错误
**现象**: AI回答"部件4在部件5的上游"，正确答案应该是"部件4在部件5的左侧"。

**原因**: VLM用了功能关系（上游/下游）描述位置，而不是视觉空间位置（左/右）。

### 2.5 问题5：LLM幻觉
**现象**: chunk里正确写的是"配气带孔盘6，6'，6''"，但AI回答时说成"部件4、5、6"。

**原因**: LLM没有严格基于检索到的chunk回答，自己编造了部件编号。

### 2.6 问题6：多个VLM描述冲突
**现象**: 同一张图片生成了多个VLM描述，一个正确一个错误，LLM有时会选错。

**原因**: 解析过程中生成了多个VLM描述chunk，没有清理错误的。

## 三、解决方案

### 3.1 解析模式选择
**方案**: 使用"General"（naive）模式替代"one"、"paper"、"book"模式。

**原因**: 
- one模式：整个文档1个chunk，检索没意义
- paper/book模式：OCR对特殊字符处理差，chunk切得太碎
- General模式：按token数切chunk，不依赖OCR识别结构

### 3.2 VLM Prompt优化

#### 3.2.1 页面描述prompt（vision_llm_describe_prompt.md）
**新增规则**:
```
## CRITICAL: FLOW DIRECTION DESCRIPTION
If the image contains a flow direction arrow (e.g., arrow labeled 7):
- Your FIRST sentence MUST state the EXACT ORDER of perforated disks the flow passes through.
- Format: "The flow (arrow 7) passes through perforated disk 6'', then perforated disk 6', then perforated disk 6."
- Do NOT start with inlet, conical section, or other structural parts.
- ONLY AFTER stating the disk sequence, describe the overall structural path.
```

#### 3.2.2 图片描述prompt（vision_llm_figure_describe_prompt.md）
**修改**: 将气流方向规则放在最前面（CRITICAL RULE - HIGHEST PRIORITY）

**新增规则**:
```
## CRITICAL RULE (HIGHEST PRIORITY)
If the image contains a flow direction arrow (e.g., arrow labeled 7):
- Your FIRST sentence MUST state the EXACT ORDER of perforated disks the flow passes through.
- Format: "The flow (arrow 7) passes through perforated disk 6'', then perforated disk 6', then perforated disk 6."
- Do NOT start with inlet, conical section, or other structural parts.
- ONLY AFTER stating the disk sequence, describe the overall structural path.
```

**修改规则**:
```
- For technical diagrams with labeled parts, explicitly describe the VISUAL SPATIAL position (left/right/above/below), NOT functional relationships like upstream/downstream.
```

### 3.3 对话System Prompt优化

**完整prompt**:
```
你是一个智能助手，请总结知识库的内容来回答问题。

重要提示：
1. 文件第7页的图片是图1（Fig.1），第8页的图片是图2（Fig.2）
2. 当问题涉及"第7页图片"或"图示"时，必须优先参考图1和图2的内容
3. 涉及部件位置、气流方向、尺寸标注等问题时，先看图片描述，再看文字内容
4. 图片中明确标注的信息（如部件编号、箭头方向、尺寸符号）以图片为准
5. 当问题问"某部件相对于某部件在图片中的位置关系"时，必须回答水平方向的位置（如"在左侧"、"在右侧"），不要回答功能关系（如"上游"、"下游"）
6. 当问题问气流方向经过哪个部件时，直接回答格式："先经过XX部件，再经过XX部件"，不要解释原因

以下是知识库：
{knowledge}
以上是知识库。
```

### 3.4 数据清理
**操作**: 删除ES中错误的VLM描述chunk，只保留正确的。

**原因**: 同一张图片可能生成多个VLM描述（一个正确一个错误），LLM可能会选错。

### 3.5 Docker配置优化
**操作**: 在docker-compose.yml中添加prompt文件的volume mount。

**原因**: 修改宿主机的prompt文件后，需要挂载到容器内才能生效。

## 五、关键教训

### 5.1 Docker容器文件修改
**教训**: 改了宿主机源码目录的文件，但容器用的是镜像内部文件，两边不互通。

**解决**: 要么用`docker cp`复制文件到容器，要么在docker-compose.yml中添加volume mount。

### 5.2 VLM Prompt优先级
**教训**: VLM prompt中的规则顺序很重要，应该把最重要的规则放在最前面。

**解决**: 将气流方向规则放在prompt的最前面（CRITICAL RULE - HIGHEST PRIORITY）。

### 5.3 多个VLM描述冲突
**教训**: 同一张图片可能生成多个VLM描述，一个正确一个错误，LLM可能会选错。

**解决**: 解析完成后，检查ES中的VLM描述chunk，删除错误的描述。

### 5.4 LLM幻觉问题
**教训**: LLM有时会自己编造信息，不严格基于检索到的chunk回答。

**解决**: 在system prompt中明确要求LLM基于知识库内容回答，不要添加知识库中没有的信息。

### 5.5 检索配置调优
**教训**: 修改检索配置（top_n、相似度阈值、向量权重、reranker）可能会导致结果变差。

**解决**: 保持默认配置，专注于优化prompt和chunk质量。
