# Embedding 模型微调 — 实现文档

## 1. 项目概述

**目标**：对 `BAAI/bge-base-zh-v1.5` 进行领域微调，提升银行-保险领域 RAG 检索效果。

**数据源**：Milvus 向量数据库中的 22,133 条文档 chunk（来自 RAG 项目 `/mnt/d/Desktop/rag-hermes/`）。

**产出物**：
- 微调后的 embedding 模型
- 训练数据集
- 微调前后评估对比

## 3. 数据准备

### 3.1 数据导出

从 Milvus `ccf_chunks` 集合导出全部 22,133 条 chunk。由于 Milvus offset 上限 16,384，采用 id 范围分批查询（batch_size=2000）。

### 3.2 训练数据生成

**流程**：
1. 从 22,133 条 chunk 中随机抽样 8,000 条（`MAX_CHUNKS=8000`，加速 LLM 生成）
2. 对每个 chunk 用 DeepSeek API 生成 2 个查询问题（10 线程并发）
3. 用 BM25 检索为每个 query 挖 3 个 hard negative（top_k=20，排除文本重叠 >60% 的候选）
4. 组装为 `{query, pos, neg}` 格式

**产出**：
- 总训练样本：14,321 条
- 训练集：12,888 条（90%）
- 测试集：1,433 条（10%）
- 评估语料库：22,133 条（全量 chunk）
- 评估测试 query：1,592 条

**脚本**：`scripts/prepare_data.py`

### 3.3 数据格式

**训练数据** (`train.jsonl`)：
```json
{"query": "什么是万能险？", "pos": ["万能险是一种..."], "neg": ["分红险的特点...", "..."]}
```

**评估语料** (`test_corpus.jsonl`)：
```json
{"docid": "12345", "text": "万能险是一种..."}
```

**评估查询** (`test_queries.jsonl`)：
```json
{"query": "什么是万能险？", "positive_passages": [{"docid": "12345", "text": "..."}]}
```

## 4. 模型微调

### 4.1 训练参数

| 参数 | 值 |
|------|-----|
| 基座模型 | `BAAI/bge-base-zh-v1.5` |
| 损失函数 | `MultipleNegativesRankingLoss` |
| Epochs | 3 |
| Batch Size | 8 |
| Learning Rate | 2e-5 |
| Warmup Ratio | 10% |
| 总步数 | 2,685 |
| 训练时长 | ~61 分钟 |

### 4.2 训练过程

| 阶段 | Loss |
|------|------|
| Step 10 (初始) | 1.6726 |
| Step 50 | 0.6275 |
| Step 500 | ~0.15 |
| Step 2680 (最终) | 0.0146 |

Loss 从 1.67 下降到 0.015，收敛良好。

### 4.3 实现细节

使用自定义训练循环（`scripts/train.py`），而非 `SentenceTransformerTrainer`，原因是 FlagEmbedding 版本兼容问题。

**关键代码**：
```python
train_loss = losses.MultipleNegativesRankingLoss(model)
# 自定义 DataLoader + 手动 forward/backward
for batch in progress:
    features_a = model.tokenize(texts_a)
    features_b = model.tokenize(texts_b)
    loss = train_loss.forward(sentence_features=[features_a, features_b], labels=None)
    loss.backward()
    optimizer.step()
```

**模型保存路径**：`output/bge-base-zh-v1.5-finetuned/`

## 5. 评估

### 5.1 评估方法

- **语料库**：22,133 条 chunk
- **测试集**：1,592 条 query（每条有标注的正例 passage）
- **指标**：Recall@10、NDCG@10
- **方法**：对 corpus 和 query 分别编码为 768 维向量，计算余弦相似度，取 top-10

**脚本**：`scripts/evaluate.py` + `scripts/run_eval_comparison.sh`

### 5.2 评估结果

| 指标 | 基座模型 | 微调模型 | 提升 |
|------|----------|----------|------|
| Recall@10 | 0.5741 | **0.8078** | **+40.7%** |
| NDCG@10 | 0.4196 | **0.6034** | **+43.8%** |

**结论**：微调后检索效果显著提升。Recall@10 从 57% 提升到 81%，意味着 top-10 结果中能命中更多相关文档。NDCG@10 提升 44%，说明排序质量也明显改善。

### 5.3 评估结果文件

- `output/eval_results/base_model.json` — 基座模型评估
- `output/eval_results/finetuned_model.json` — 微调模型评估

## 7. 踩坑记录

1. **Milvus offset 上限 16,384**：不能用 `offset` 翻页，改用 `id > last_id` 范围查询
2. **HF 不认中文路径**：模型路径不能含中文，放在 `~/` 下
3. **torch 版本**：transformers ≥4.57 需要 torch ≥2.6（CVE 修复），CPU 版会导致 sentence_transformers 回退 CPU
4. **tf_keras mock 坑**：py3.11 里真实 tf_keras 能用（只有警告），不要 mock 它
5. **WSL2 训练路径**：不能放 `/mnt/d/` 下（Bus error），要放在 `~/`
6. **SentenceTransformerTrainer 兼容性**：FlagEmbedding 1.4.0 的 Trainer 与 transformers 4.57 有兼容问题，改用自定义训练循环
