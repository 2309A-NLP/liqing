## 0. 一句话

实时交互流式数字人引擎：**文字/语音输入 →（可选 LLM）→ TTS 合成语音 → ASR 提声学特征 → 口型模型推理 → 贴回高清视频 → WebRTC/RTMP/虚拟摄像头推流**。Apache 2.0。

---

## 1. 整体数据流（最重要，务必理解）

```
[浏览器/API]
   │  POST /human {text,type:echo|chat}   或   POST /humanaudio {file}
   ▼
[server/routes.py]  human()/humanaudio()
   │  echo→直接put_msg_txt ; chat→llm_response()后台线程
   ▼
[llm.py] (chat模式) 通义 qwen-plus 流式 → 按标点切句 → put_msg_txt()
   ▼
[tts/*] BaseTTS.msgqueue → process_tts线程 → txt_to_audio()
   │  合成音频 → 重采样16k → 切320样本/帧(20ms) → parent.put_audio_frame()
   ▼
[avatars/audio_features/*ASR] BaseASR.queue  (注意：这里的"ASR"是"音频特征提取"，不是语音识别!)
   │  run_step(): 攒帧 → 提声学特征(Mel/Whisper) → feat_queue
   ▼
[avatars/base_avatar.py]  三线程流水线（核心）
   │  ① inference():   feat_queue → inference_batch() → res_frame_queue
   │  ② process_frames(): res_frame_queue → paste_back_frame()贴回 → output.push_*
   │  ③ render():      主循环驱动 asr.run_step() + 限流
   ▼
[streamout/*]  webrtc / rtmp / virtualcam
   ▼
[server/webrtc.py] HumanPlayer/PlayerStreamTrack → 浏览器
```

**关键澄清：项目里有两个"ASR"概念，别混淆**
- `avatars/audio_features/`（MelASR/whisper）= **音频→声学特征**，喂给口型模型，每个会话必有。
- `server/asr_server.py`（SenseVoice/FunASR）= **真正的语音识别(语音→文字)**，给语音输入用，可选，走 WebSocket `/api/asr`。

## 2. 三线程流水线（avatars/base_avatar.py，核心中的核心）

`render(quit_event)` 起三条线程，用 Queue 解耦，生产者-消费者：

| 线程 | 函数 | 输入队列 | 输出队列 | 干啥 |
|---|---|---|---|---|
| 主循环 | `render` | — | — | 循环调 `asr.run_step()` 提特征；按 `output.get_buffer_size()` 积压**限流**(sleep) |
| 推理 | `inference` | `asr.feat_queue` | `res_frame_queue` | 取一批特征→`inference_batch()`；**静音帧不推理只取原图**(省GPU) |
| 渲染 | `process_frames` | `res_frame_queue` | `output` | `paste_back_frame()`把口型贴回高清原图→推帧 |

**省 GPU 的设计**：静音(type≠0)时直接用 `frame_list_cycle[idx]` 原始帧，只有说话时才跑模型。`mirror_index()` 让待机视频来回循环播放不跳变。

**打断机制**：`flush_talk()` 清空 TTS msgqueue + ASR queue + state=PAUSE，实现说话中途打断。API: `/interrupt_talk` 或 `/human {interrupt:true}`。

## 3. 插件注册表（registry.py，扩展的入口）

去中心化注册，5 类：`stt / llm / tts / avatar / output`。

```python
@register("tts", "edgetts")
class EdgeTTS(BaseTTS): ...
# 创建：registry.create("tts", "edgetts", opt=opt, parent=self)
```

`app.py` 按 `--model` 用 `importlib` **动态导入**对应 avatar 模块（只加载需要的，避免全部依赖）。TTS/output 同理在 `base_avatar.__init__` 里按 `opt.tts`/`opt.transport` 动态导入。

**新增插件三步**：① 写类继承 BaseXxx ② `@register("类别","名字")` ③ 在对应 `_xxx_modules` 字典里加映射。

## 4. 三个口型模型对比

| 模型 | 文件 | 特征提取 | 显卡要求 | 质量 |
|---|---|---|---|---|
| **wav2lip** | `avatars/wav2lip_avatar.py` | Mel频谱(`audio_features/mel.py`) | RTX3060+ 实时 | 一般，256分辨率 |
| **musetalk** | `avatars/musetalk_avatar.py` | Whisper | RTX3080Ti+ 实时 | 高 |
| **ultralight** | `avatars/ultralight_avatar.py` | — | 最轻 | 轻量 |

每个 avatar 类只需实现两个方法：`inference_batch(index, audiofeat_batch)`（推理）和 `paste_back_frame(pred, idx)`（贴回）。

形象数据放 `data/avatars/<avatar_id>/`：`full_imgs/`(高清原帧) + `face_imgs/`(裁切人脸) + `coords.pkl`(人脸坐标)。

## 5. TTS 层（tts/，声音克隆相关）

`BaseTTS`：msgqueue + process_tts 线程 + `txt_to_audio()` 待实现。已支持：edgetts(默认,免费在线)、gpt-sovits、cosyvoice、fishtts、tencent、doubao、indextts2、azuretts、qwentts、omnitts。

- **edgetts** 最简单，无需 key，`REF_FILE` 是音色名(如 `zh-CN-YunxiaNeural`)。
- 声音克隆走 `REF_FILE`+`REF_TEXT`(参考音频+文本)，offer 时可传 `refaudio`/`reftext` 按会话定制。
- 切句逻辑在 llm.py：流式输出按标点 `,。!?` 切 >10 字就送 TTS，降低首字延迟。

## 6. API 一览（docs/api.md）

统一返回 `{code,msg,data}`，code=0 成功。

| 接口 | 方法 | 作用 |
|---|---|---|
| `/offer` | POST | WebRTC 建连，返回 sessionid。可带 avatar/refaudio/reftext/custom_config |
| `/human` | POST | 文本驱动。type=echo(复读)/chat(LLM)；interrupt 打断；tts 透传音色/情感 |
| `/humanaudio` | POST | 上传音频文件直接播 (multipart) |
| `/interrupt_talk` | POST | 打断 |
| `/is_speaking` | POST | 查是否在说话 |
| `/record` | POST | start_record/end_record 服务端录制 |
| `/record/{sessionid}` | GET | 下载 MP4 |
| `/set_audiotype` | POST | 动作编排（不说话时播自定义视频/动作） |
| `/api/asr` | WS | 本地语音识别(SenseVoice，装了 funasr 才有) |
| `/api/admin/config`,`/sessions` | GET | 后台监控 |

Web 页面：`/index.html`(主交互)、`/avatar.html`(上传视频生成形象)、`/admin.html`(后台)。

## 7. 会话与并发

- `server/session_manager.py`：单例，`sessions: {sessionid: BaseAvatar}`，`max_session` 限流，UUID 作 sessionid。建会话时在线程池加载（模型耗时）。
- `server/rtc_manager.py`：管 PeerConnection 生命周期，连接断开自动 `remove_session`。
- 并发瓶颈：不说话看 CPU，同时说话看 GPU。`inferfps`+`finalfps` 都要 ≥25 才算实时。

## 8. 本机环境可行性（2026-06-22 实测）

| 项 | 状态 | 影响 |
|---|---|---|
| **GPU** | ✅ **RTX 4060, 8GB 显存**（驱动596.36/CUDA13.2，WSL2 直通正常，nvidia-smi 在 `/usr/lib/wsl/lib/`） | **wav2lip256 可实时(>60fps预期)**；musetalk 可跑但 8GB 偏紧需控 batch |
| conda | ❌ 未装 | 需先装 miniconda 或用系统 py3.12 + venv |
| torch | ❌ 未装 | 需装 **CUDA 版**（cu12x/cu13，匹配驱动） |
| Python | ✅ 3.12.3 | 满足 3.10+ |
| ffmpeg | ✅ 6.1.1 | 录制/合成可用 |
| 模型权重 | ❌ 未下载 | `models/` 只有占位符，需从夸克/GDrive 下 wav2lip.pth |
| 形象数据 | ❌ 未下载 | `data/avatars/` 只有 .gitkeep，需下 wav2lip256_avatar1 |
| **沙箱** | ⚠️ 杀后台服务、无无头浏览器 | app.py 需以后台服务跑(沙箱会杀)；WebRTC 看画面要用户自己的浏览器开 |

**结论（重要，已纠正）**：本机有 **RTX 4060**，**真机实时 demo 完全可行**(wav2lip256 在 4060 上稳超 25fps)。可行路径：
1. **优先真机跑通** wav2lip 全链路(装 CUDA torch + 下模型/形象 → 启 app.py → 浏览器看数字人说话)。
2. **显存约束**：4060 是 8GB，wav2lip256 富余；若用 musetalk 要留意显存，必要时调小 `--batch_size`。
3. **沙箱注意**：`app.py` 要用 `run_in_background` 起；WebRTC 画面需用户在 Windows 浏览器开 `http://localhost:8010/index.html`(WSL 端口转发)，截图由用户配合或我驱动 API 后让用户看。
4. 不依赖 GPU 的逻辑(TTS切句/session/API/插件/RAG)照样能单测。

## 10. 快速启动备忘（有 GPU 时）

```bash
conda create -n livetalking python=3.12 && conda activate livetalking
pip install torch==2.9.1 torchvision torchaudio --index-url https://download.pytorch.org/whl/cu130
pip install -r requirements.txt
# 下 wav2lip.pth → models/ ; 解压形象 → data/avatars/
python app.py --transport webrtc --model wav2lip --avatar_id wav2lip256_avatar1
# 浏览器 http://localhost:8010/index.html
```
