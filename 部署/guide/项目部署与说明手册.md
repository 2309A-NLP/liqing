# 项目部署与说明手册

本文档用于统一说明本项目的部署方式、目录结构、核心配置、性能优化思路以及模型职责，方便快速上手和交付说明。

---

## 一、项目概述

本项目是一个基于 **FastAPI + Milvus + Redis + DeepSeek** 的中文多角色 RAG 系统，支持：

- 向量检索
- BM25 关键词检索
- rerank 重排序
- 短期会话记忆
- 长期记忆
- 多角色提示词
- 前端检索可视化
- 性能统计面板

## 五、核心运行流程

1. 用户输入问题
2. `BGEM3FlagModel` 将问题转成向量
3. Milvus 做向量召回
4. BM25 做关键词召回
5. `FlagReranker` 对候选结果精排
6. DeepSeek LLM 根据上下文生成答案
7. 前端展示答案、检索结果、性能指标和 prompt 预览

## 六、性能优化摘要

本项目近期做过的核心优化包括：

- 缩短 embedding 输入长度
- 调低 dense 检索参数
- 缓存 BM25 索引
- 并发执行 dense / BM25 / memory 召回
- 收缩候选数量
- 控制上下文长度
- 截断 prompt 预览
- 增加性能埋点
- 前端展示性能统计面板
- 增加基准测试脚本

详细说明见 `RAG性能优化说明.md`。

## 七、模型职责说明

项目中各模型/组件的职责如下：

- `BGEM3FlagModel`：文本向量化，负责语义召回入口
- `BM25Okapi`：关键词召回，负责补充精确匹配
- `FlagReranker`：候选精排，负责筛选最相关文本
- DeepSeek LLM：最终回答生成
- `RoleConfig`：角色风格和规则模板
- `RetrievedDoc`：检索结果数据结构
- `RetrievalStats`：检索过程统计信息

详细说明见 `模型作用说明.md`。

## 九、常见排错

### 1. 相对导入报错
使用：

```bash
# 从项目根目录运行：

```

### 2. 依赖不完整
重新安装：

```bash
pip install -r requirements-lock.txt
```

### 3. Redis / Milvus 连接失败
检查：
- 服务是否启动
- 地址是否正确
- 用户名密码是否匹配

### 4. 模型加载失败
检查：
- `EMBED_MODEL_PATH`
- `RERANK_MODEL_PATH`
- DeepSeek API Key 是否正确
